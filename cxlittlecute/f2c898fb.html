Elasticsearch（二）-镜像下载启动

本文主要记录自己在docker安装Elasticsearch时出现的问题，以及解决方式。

下载镜像

docker pull elasticsearch:7.4.2 # 存储和检索数据

挂载配置文件

在挂载配置文件的时候可能会出现两个问题：

  1. 虚拟机中的文件权限不足。
  2. 挂载的文件路径错误。

解决问题：
问题一：修改虚拟机中挂载的文件权限。
问题二：核对挂载的文件路径是否正确。

mkdir -p /mydata/elasticsearch/config # 在mydata文件夹下创建es的config文件夹，将docker中es的配置挂载在外部，当我们在linux虚拟机中修改es的配置文件时，就会同时修改docker中的es的配置
mkdir -p /mydata/elasticsearch/data #在mydata文件夹下创建es的data文件夹
echo "http.host: 0.0.0.0" >> /mydata/elasticsearch/config/elasticsearch.yml # [http.host: 0.0.0.0]允许任何远程机器访问es，并将其写入es的配置文件中

chmod -R 777 /mydata/elasticsearch/ # 保证权限问题

启动容器：

docker run --name elasticsearch -p 9200:9200 -p 9300:9300 \
-e "discovery.type=single-node" \
-e ES_JAVA_OPTS="-Xms64m -Xmx512m" \
-v /mydata/elasticsearch/config/elasticsearch.yml:/usr/share/elasticsearch/config/elasticsearch.yml \
-v /mydata/elasticsearch/data:/usr/share/elasticsearch/data \
-v /mydata/elasticsearch/plugins:/usr/share/elasticsearch/plugins \
-d elasticsearch:7.4.2

# docker run --name elasticsearch 创建一个es容器并起一个名字；
# -p 9200:9200 将linux的9200端口映射到docker容器的9200端口，用来给es发送http请求
# -p 9300:9300 9300是es在分布式集群状态下节点之间的通信端口  \ 换行符
# -e 指定一个参数，当前es以单节点模式运行
# *注意，ES_JAVA_OPTS非常重要，指定开发时es运行时的最小和最大内存占用为64M和128M，否则就会占用全部可用内存
# -v 挂载命令，将虚拟机中的路径和docker中的路径进行关联
# -d 后台启动服务

主要问题

在我启动容器后，没有出现报错，但是使用命令docker ps，发现容器并没有启动成功，也就是没有正常运行。

解决：

使用docker logs -f 容器ID查看容器运行的日志。

发现出现以下问题：

OpenJDK 64-Bit Server VM warning: Option UseConcMarkSweepGC was deprecated in version 9.0....

这是由于jdk版本太低造成的，jdk版本需要在9.0以上，因此需要重新安装jdk。重新安全JDK后就可以正常启动了。
还有就是允许任何远程机器访问es中的http.host: 0.0.0.0是有空格的，如果没有空格它也会报错elasticsearch.yml有问题。

使用IP+端口访问es,出现下图说明安装成功：

Kibana

最后我们也可以安装es的可视化界面-kibana。

docker pull kibana:7.4.2
docker run --name kibana -e ELASTICSEARCH_HOSTS=http://47.106.192.255:9200 -p 5601:5601 \
-d kibana:7.4.2

注意修改将IP和端口，修改对应为你自己ES的IP和端口。
