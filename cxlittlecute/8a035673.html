基本数据类型 包装类 String之间的相互转换

类型转换是我们经常会遇到的问题，今天通过本博客进一步总结它的用法。

  • Java提供了8种基本数据类型对应的包装类，使得基本数据类型的变量具有类的特征。
  • 基本数据类型转变为了包装类，就可以调用类中的方法。

注意：除了Boolean和Character(没有父类)其他包装类的父类都是Number，我们可以根据以下的源码看出：

基本数据类型->包装类

  • 调用包装类的构造器
  • 通过字符串参数
  • 自动装箱

public void test11(){
    // 调用包装类的构造器
    int num1 = 10;
    Integer integer1 = new Integer(10);
    System.out.println(integer1.toString());

    //  通过字符串参数
    Integer integer2 = new Integer("123");
    System.out.println(integer2.toString());

}

但是需要注意，使用字符串转换的时候，如果是以Number为父类的包装类，使用字符串的构造器，里面的参数如果是又有数字又有字母会报错(也就是说参数想转什么类型，参数就必须符合这种结构)，但是Boolean支持这样，只会判断为false。

public void test11(){
    Integer integer3 = new Integer("asb123"); // 报错，参数必须对应包装类的类型
    System.out.println(integer3.toString());

    Boolean asb123 = new Boolean("asb123");
    System.out.println(asb123); // 不会报错，结果为false

}

为什么会这样叻？
我们来看看Boolean的源码:

public Boolean(String var1) {
    this(parseBoolean(var1));
}

public static boolean parseBoolean(String var0) {
    return var0 != null && var0.equalsIgnoreCase("true");
}

在忽略大小写的情况下如果是true那么就是true，如果不是同事不为null，那么都为false。

包装类->基本数据类型

首先，我们要知道这个操作也是必要的，因为我们的包装类属于类是不能进行加减乘除运算的。

  • 调用包装类的xxxValue（），你是哪个包装类型就调用哪个value()方法.
  • 自动拆箱
public void test12(){
    Integer in1 = new Integer(12);
    int i = in1.intValue();
    System.out.println(i);
}

自动装箱和自动拆箱

那么说到这里就得说一下自动装箱和自动拆箱了，它属于Jdk5的新特性。我们通过下面的例子理解这个特性：

public void test14(){
    int num = 20;
    test13(num);
}

public void test13(Object o){
    System.out.println(o);
}

上面的这个操作将num放入test13中发现是可以的，这里并不是说我们把Object obj = num,这样理解是不对的，因为int类型和Object类型是没有任何关系的，但为什么又行了？

因为这里就用到了自动装箱,自动装箱就是这样的,相反有的就是自动拆箱。

  • 自动装箱：基本数据类型->包装类
  • 自动拆箱：包装类->基本数据类型
public void test15(){
    // 自动装箱
    int i = 10;
    Integer integer = i;
    // 自动拆箱
    int i2 = integer;
}

基本数据类型、包装类->String

  • 方式一：连接运算
  • 方法二：调用String重载的valueOf（Xxx xx）
    public void test16(){
    // 方式一：
    int num1 = 10;
    String str1 = num1 + "";
    // 方式二：
    float f1 = 12.3f;
    String str2 = String.valueOf(f1);
    
}

String->基本数据类型、包装类

  • 调用包装类的parseXxx（String x）
public void test17(){
    String str = "123";
    int i = Integer.parseInt(str);
}

注意：
最后再强调一下，字符串转换的时候如果有数字和字母，那么会报错NumberFormatException，除了Boolean特殊一些，可以再看一下上面的源码解析。

同时因为基本数据类型和包装类两个之间有自动装箱和自行拆箱的特性，所以我们可以把他们两个相对string来说看成一个整体进行转换。
