RocketMQ(四)-事务消息

本章主要围绕以下内容展开：

  • 1、什么是事务消息
  • 2、事务消息流程执行图
    • 2.1、事务消息的发送及提交
    • 2.2、事务补偿
    • 2.3、事务消息状态

1、什么是事务消息

事务消息就是说生产者发送的消息是有事务控制的，在生产者没有提交消息之前，消费者是不会消费到这些未提交的消息的（短暂对消费者不可见）。

2、事务消息流程执行图

大体上是两个流程：

  • 正常事务消息的发送及提交
  • 事务消息的补偿流程（消息队列的回查）

2.1、事务消息的发送及提交

  • (1)发送消息（half消息）
  • (2)服务端响应消息
  • (3)根据发送结果执行本地消息（如果写入失败，此时half消息对业务不可见，本地逻辑不执行）
  • (4)根据本地事务状态执行Commit或者Rollback（Commit操作生成消息索引，消息对消费者可见）

2.2、事务补偿

  • (1) 对没有Commit/Rollback的事务消息（pending状态的消息），从服务端发起一次“回查”

  • (2) Producer收到回查消息，检查回查消息对应的本地事务的状态

  • (3) 根据本地事务状态，重新Commit或者Rollback

    注意：其中，补偿阶段用于解决消息Commit或者Rollback发生超时或者失败的情况。

2.3、事务消息状态

事务状态共有3种：

  • 提交状态
  • 回滚状态
  • 中间状态

代码表现形式：

  • TransactionStatus.CommitTransaction: 提交事务，它允许消费者消费此消息。
  • TransactionStatus.RollbackTransaction: 回滚事务，它代表该消息将被删除，不允许被消费。
  • TransactionStatus.Unknown: 中间状态，它代表需要检查消息队列来确定状态。

3、事务消息的使用

3.1、事务生产者

3.1.1、创建事务消息生产者

//1.创建消息生产者producer，并制定生产者组名
TransactionMQProducer producer = new TransactionMQProducer("group5");

3.1.2、添加事务监听器

//添加事务监听器
producer.setTransactionListener(new TransactionListener() {
    /**
        * 在该方法中执行本地事务
        * @param msg
        * @param arg
        * @return
        */
    @Override
    public LocalTransactionState executeLocalTransaction(Message msg, Object arg) {
        if (StringUtils.equals("TAGA", msg.getTags())) {
            return LocalTransactionState.COMMIT_MESSAGE;
        } else if (StringUtils.equals("TAGB", msg.getTags())) {
            return LocalTransactionState.ROLLBACK_MESSAGE;
        } else if (StringUtils.equals("TAGC", msg.getTags())) {
            return LocalTransactionState.UNKNOW;
        }
        return LocalTransactionState.UNKNOW;
    }

    /**
        * 该方法时MQ进行消息事务状态回查
        * @param msg
        * @return
        */
    @Override
    public LocalTransactionState checkLocalTransaction(MessageExt msg) {
        System.out.println("消息的Tag:" + msg.getTags());
        return LocalTransactionState.COMMIT_MESSAGE;
    }
});

3.1.2、发送事务消息

// 可以指定某一条消息进行事务控制，也可以对所有消息进行事务控制
SendResult result = producer.sendMessageInTransaction(msg, null);
